Ensure your pull request adheres to the following guidelines:

Search the current list and pull requests to avoid adding duplicates.<br>
Make sure the software project is developed open-source.<br>
List each software project under the correct category.<br>
Please adhere to the alphabetical ordering of the list.<br>
Each project should go under a subheading with the respective programming language.<br>
Use the following format: [Name](link) - Description/Link.<br>
Where Description/Link should be a very short summary of the project.<br>
Try to keep each entry (name + description/link) to one line.<br>
Start the description with a capital and end with a full stop.<br>
Don't start the description with A or An.<br>
New categories or improvements to the existing categorisation are welcome.<br>
Check your spelling and grammar.<br>
You can also add related links and repositories in the end.<br>
